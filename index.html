What is SASS?

CSS on its own can be fun, but stylesheets are getting larger, more complex, and harder to maintain.

Sass is a preprocessor that lets you use features that don't exist in CSS yet like variables, nesting, mixins, inheritance and other goodies that make writing CSS fun again.

How does it work?

  1. You write Sass (.scss files)
  2. You run a command (sass --watch) on the command line
  3. .scss files compile into .css files

How some people feel about SASS...

"I write stylesheets by hand! I don’t need help! And I certainly don’t want to add extra complexity to my workflow. Go away!"

The reality...

  • Sass is extremely powerful
  • Sass increases efficency
  • Sass makes your life easier*

*If not implemented correctly, this is not guarenteed.

Variables

Think of variables as a way of storing information that you want to reuse throughout your stylesheet.

You can store things like colors, font stacks, or any CSS value you'll want to reuse.

Example

Sass uses the "$" symbol to make something a variable:


$font-stack:    Helvetica, sans-serif;
$primary-color: #333;

body {
  font: 100% $font-stack;
  color: $primary-color;
}

CSS


body {
  font: 100% Helvetica, sans-serif;
  color: #333;
}

Nesting

Unlike HTML, CSS does not have a clear visual hierarchy.

Sass will let you nest your CSS selectors in a way that follows the same visual hierarchy of HTML.

* Be aware, overly nested rules could make your code hard to maintain.

Example


nav {
  ul {
    margin: 0;
    padding: 0;
    list-style: none;
  }

  li { display: inline-block; }

  a {
    display: block;
    padding: 6px 12px;
    text-decoration: none;
  }
}

CSS


nav ul {
  margin: 0;
  padding: 0;
  list-style: none;
}

nav li {
  display: inline-block;
}

nav a {
  display: block;
  padding: 6px 12px;
  text-decoration: none;
}

Partials

You can create a partial Sass file that can contain little snippets of CSS that you can include in other SASS files.

This is a fantastic way to modularize your CSS, making maintenance easier.

Example

A partial is simply a Sass file named with a leading underscore:


_partial.scss

The underscore lets Sass know that the file is only a partial file and it should not be generated into a CSS file.

Import

Sass partials are used with the "@import" directive.

Let's say you have a couple of Sass files, _reset.scss and main.scss. We want to import _reset.scss into main.scss

Example


// _reset.scss

html,
body,
ul,
ol {
   margin: 0;
  padding: 0;
}
					

// main.scss

@import 'reset';

body {
  font: 100% Helvetica, sans-serif;
  background-color: #efefef;
}

CSS


html, body, ul, ol {
  margin: 0;
  padding: 0;
}

body {
  font: 100% Helvetica, sans-serif;
  background-color: #efefef;
}

Mixins

Some things in CSS are a bit tedious to write, especially with CSS3 and the many vendor prefixes that exist.

A mixin lets you make groups of CSS declarations that you want to reuse throughout your site. You can even pass in values to make your mixin more flexible.

Example

To create a mixin you use the @mixin directive and give it a name. We're also using the variable $radius inside the parentheses so we can pass in a radius of whatever we want. 


@mixin border-radius($radius) {
  -webkit-border-radius: $radius;
     -moz-border-radius: $radius;
      -ms-border-radius: $radius;
          border-radius: $radius;
}

.box { 
	@include border-radius(10px); 
}

After you create your mixin, you can then use it as a CSS declaration starting with @include followed by the name of the mixin.

CSS


.box {
  -webkit-border-radius: 10px;
  -moz-border-radius: 10px;
  -ms-border-radius: 10px;
  border-radius: 10px;
}

Inheritance

This is one of the most useful features of Sass. Using "@extend" lets you share a set of CSS properties from one selector to another.

It helps keep your Sass very DRY. In our example, we're going to create a simple series of messaging for errors and successes.

Example


.message {
  border: 1px solid #ccc;
  padding: 10px;
  color: #333;
}

.success {
  @extend .message;
  border-color: green;
}

.error {
  @extend .message;
  border-color: red;
}

CSS


.message, .success, .error {
  border: 1px solid #cccccc;
  padding: 10px;
  color: #333;
}

.success {
  border-color: green;
}

.error {
  border-color: red;
}

Operators

Doing math in your CSS is very helpful.
Sass allows you to do that.


.container { width: 100%; }

article[role="main"] {
  float: left;
  width: 600px / 960px * 100%;
}

aside[role="complimentary"] {
  float: right;
  width: 300px / 960px * 100%;
}

We've created a very simple fluid grid, based on 960px. Operations in Sass let us do something like take pixel values and convert them to percentages without much hassle.

CSS


.container {
  width: 100%;
}

article[role="main"] {
  float: left;
  width: 62.5%;
}

aside[role="complimentary"] {
  float: right;
  width: 31.25%;
}

Compass

Compass is an open-source CSS authoring framework which uses the Sass stylesheet language to make writing stylesheets powerful and easy.

http://compass-style.org/

Before we continue...

It's important to know that Sass has two syntaxes:

  • SCSS: Main syntax ("Sassy CSS"). New as of Sass 3
  • Sass: Older syntax

* All examples we saw today were written in "SCSS" format.

Best practices

  • Avoid nesting too deep. Nesting is great for components (i.e. header, footer, nav, etc.)
  • Use hyphenated class names (no camelCase or snake_case).
  • Name variables modularly.
  • Structure your partial files so it's easy to maintain: https://github.com/lvecchio/sass-project-structure/tree/master/sass
